# Stack Size Config（stacksizeconf）使用说明

## 模组简介

本模组在 **NeoForge** 环境下，为 Minecraft 中的物品提供**可配置的堆叠上限**。效果适用于各类物品（包括原版中原本不可堆叠的物品）。  
**合并堆叠时仍要求物品数据一致**（例如附魔、耐久、组件等需相同，与原版逻辑一致）。

---

## 环境与依赖

| 项目 | 说明 |
|------|------|
| Minecraft | 1.21.11 |
| 模组加载器 | NeoForge（版本需满足模组声明的范围，见 `neoforge.mods.toml`） |
| 安装位置 | 将构建得到的 `.jar` 放入游戏目录下的 `mods` 文件夹 |

单机与多人游戏中，**服务端与客户端均需安装**且配置一致，避免出现不同步问题。

---

## 如何打开配置

任选其一：

1. **游戏内**：主菜单 → **Mods（模组）** → 找到 **Stack Size Config** → **Config（配置）**  
2. **直接编辑文件**：游戏配置目录下的 `config/stacksizeconf-common.toml`  
   - 单机通常为 `.minecraft/config/stacksizeconf-common.toml`  
   - 修改后一般需**重启游戏**或按 NeoForge 提示重载（以实际版本行为为准）

---

## 配置项说明

### `enableStackSizeOverrides`（启用自定义堆叠）

- **默认**：`true`  
- **作用**：为 `false` 时**完全使用原版**堆叠规则，本模组不再改写上限。

### `stackSizeMultiplier`（堆叠倍率）

- **默认**：`2.0`  
- **范围**：`0.05`～`1024.0`  
- **作用**：在下列「基准数量」上乘以该倍率（四舍五入取整），再受全局封顶限制。  
  - 原版可堆叠物品：基准为**该物品在原版中的最大堆叠数**（例如 64）。  
  - 原版不可堆叠（上限为 1）的物品：基准为 **`nonStackableBaseMax`**（见下一条）。

### `nonStackableBaseMax`（原不可堆叠物品的基准数量）

- **默认**：`64`  
- **范围**：`1`～`65536`  
- **作用**：仅对原版**最大堆叠为 1** 的物品，在参与倍率计算时使用的**基准堆叠数**。  
  - 例如倍率为 `2.0`、基准为 `64` 时，这些物品的理论上限为 `128`（再受 `maxStackHardCap` 限制）。

### `maxStackHardCap`（全局堆叠上限 / 封顶）

- **默认**：`1024`  
- **范围**：`1`～`65536`  
- **作用**：无论倍率与基准算出来多大，**最终堆叠数不会超过此值**。

### 计算逻辑（便于你预估效果）

在启用自定义堆叠时，对某物品：

1. 若原版最大堆叠 ≤ 1：基准 = `nonStackableBaseMax`（且至少为 1）。  
2. 否则：基准 = 该物品的**原版**最大堆叠数。  
3. 临时结果 = `round(基准 × stackSizeMultiplier)`，再限制在 `[1, maxStackHardCap]`。

## 使用建议与注意事项

- **合并条件**：能放进同一格仍须满足原版对「同一物品」的要求（数据一致才可合并）。  
- **平衡性**：倍率与封顶过大可能影响经济、运输与部分模组逻辑，请按需调整。  
- **联机**：服务端 `stacksizeconf-common.toml` 应与客户端一致，避免异常。  
- **关闭模组效果**：将 `enableStackSizeOverrides` 设为 `false` 即可恢复原版堆叠，无需卸载模组（若你希望完全不用本模组，再移除 jar）。

## 开发者：构建模组

在项目根目录执行（需已安装 JDK 与网络可访问 Gradle/Maven 依赖）：

```bash
./gradlew build
```

生成的模组文件一般在 `build/libs/` 下（具体文件名以构建输出为准）。

若在中国大陆网络下构建失败，可在 `gradle.properties` 中按需配置代理（项目中已有示例注释，可按本地端口修改或注释掉）。
